# Contributing

TontoJB accepts contributions that improve research clarity, publication hygiene, attribution, reproducibility notes, and responsible documentation. Contributions must stay within the repository's research-only boundaries.

## Good contribution types

- Documentation corrections, safer wording, and clearer firmware scope.
- Attribution, provenance, NOTICE, LICENSE, or DISCLAIMER improvements.
- Tests or syntax checks that improve repository quality without adding new exploitation capability.
- Redaction, ignore-rule, or hygiene improvements that prevent private artifacts from being published.
- Research notes that are scoped, reproducible, and do not expose private logs or identifiers.

## Contributions that are not accepted

- Piracy, cheating, payment evasion, DRM abuse, or unauthorized-access support.
- Payload weaponization, stealth, persistence, or online-service abuse features.
- Private certificates, runtime telemetry, raw crash dumps, local network logs, or account identifiers.
- Firmware-shopping guides or exploit reliability tuning framed as end-user jailbreak support.
- Material that claims vendor affiliation or upstream endorsement where none exists.

## Pull request checklist

Before proposing changes:

1. State the firmware and environment assumptions affected by the change.
2. Keep public docs descriptive rather than step-by-step operational guidance.
3. Preserve upstream attribution and license notices.
4. Run relevant syntax checks for changed scripts when applicable.
5. Confirm no private logs, certificates, telemetry, or local paths were staged.

By contributing, you agree that your contribution is provided under the repository license and within the responsible-use boundaries described in [`DISCLAIMER.md`](DISCLAIMER.md) and [`SECURITY.md`](SECURITY.md).
