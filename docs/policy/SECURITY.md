# Security Policy

TontoJB is maintained as a security-research proof of concept. Reports are most useful when they improve responsible publication, reduce accidental data exposure, clarify safety boundaries, or identify repository hygiene problems.

## In scope

- Accidental exposure of private keys, certificates, telemetry, logs, crash dumps, identifiers, or private network details.
- Documentation that could be misread as consumer jailbreak support, piracy enablement, unauthorized-access guidance, or vendor affiliation.
- Repository hygiene issues that cause ignored local artifacts to become publishable.
- Attribution, provenance, or licensing issues involving third-party research or reference material.
- Bugs in helper tooling that create unintended local file disclosure or unsafe logging behavior.

## Out of scope

- Requests for help running exploitation against devices or services you do not own.
- Requests for piracy, cheating, account bypass, DRM circumvention, payment evasion, or online-service abuse.
- Firmware shopping, exploit reliability tuning, payload weaponization, or operational jailbreak support.
- Reports that require attacking third-party infrastructure, accounts, consoles, networks, or services.
- Demands for support, warranty, offsets, exploit updates, or compatibility with unsupported firmware.

## Reporting guidance

If a report contains private data or sensitive details, do not paste it into a public issue. Use GitHub private vulnerability reporting if it is enabled for this repository, or request a private contact path before sharing sensitive material.

Good reports include:

- A short summary of the issue.
- The affected file or documentation section.
- Why it matters for public safety, attribution, privacy, or responsible research.
- A minimal reproduction or redacted example when safe to share.

The project is not affiliated with Sony, PlayStation, Netflix, FreeBSD, or referenced upstream researchers. Do not treat this repository as an official vendor disclosure channel.
